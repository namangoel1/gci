The State of Encrypted Web Communication

What is HTTPS?

HTTPS stands for HyperText Transfer Protocol Secure, a method of secure communication on the internet. It is formed by the layering of SSL/TLS over the Hypertext Transfer Protocol.

The security of HTTPS comes from TLS. HTTPS security provides authentication between the website and the webserver. Any data sent from the user to the webserver is encrypted, and vice versa. This prevents man-in-the-middle attacks, which exploit insecure communication between a website and it's host.

Why isn't HTTPS universal?

Technical

Enabling HTTPS on your site requires you to install a SSL Certificate from a trusted certificate authority. Few such authorities exist and they can easily exploit the market. The certificate costs from $8 to $70, and installation is not a technically challenging procedure; difficult for novice users.

Politics

Surveillance programs find it to their advantage to install backdoors in sites and to avoid HTTPS. For example, VeriSign gives user data if the law requires it.

Making HTTPS Universal

The Electronic Frontier Foundation and the Tor Project aim for a better, secure, private, and safe web.

Let's Encrypt CA aims to offer free SSL certificates, beating the main hurdle when it comes to webmasters installing SSL.

The HTTPS Everywhere browser extension forces the browser to use the encrypted protocol wherever possible.

Google recently made HTTPS a ranking signal. Sites that enable HTTPS are ranked higher in search results. This is to encourage developers and webmasters to move to HTTPS and to ensure user safety.

One Time Password (OTP) drives like the Crypto Stick allow stronger and randomized passwords. People are bad at passwords and hardware devices like this allow another secure layer.

2-step authentication which needs access to the user's mobile device as well as their password make it difficult to comprimise accounts. Google and Dropbox are some popular names that offer this.
